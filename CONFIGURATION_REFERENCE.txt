    File:    backend/app/core/config.py  Line 9
    Current: SECRET_KEY = "secret_key_for_security_should_be_changed_in_production"
    Risk:    JWT tokens can be forged if this default is used.
    Fix:     Set a cryptographically random 64-char hex string in .env

    File:    backend/app/core/config.py  Line 15
    Current: POSTGRES_PASSWORD = "postgres"
    Risk:    Database is exposed with a trivial password.
    Fix:     Use a strong password in .env: POSTGRES_PASSWORD=<strong_password>

  ⚠️  WARNING 3 — CORS allows all origins
    File:    backend/app/main.py  Line 17
    Current: allow_origins=["*"]
    Risk:    Any website on the internet can make requests to your API.
    Fix:     Change to: allow_origins=["https://yourdomain.com"]

  ⚠️  WARNING 4 — Alembic migrations NOT auto-run in Docker
    File:    docker/backend/entrypoint.sh  Lines 8-9
    Current: "alembic upgrade head" is commented out (not running)
    Risk:    Docker container starts without applying schema migrations.
    Fix:     Uncomment or add: alembic upgrade head
    OR run manually: docker exec resume_platform_backend alembic upgrade head

  ℹ️  NOTE — Default user created without real auth
    File:    backend/app/services/agent_workflow.py  Lines 227-238
    Current: If no user exists, creates "agent_recruiter@example.com"
             with hashed_password="agent_pass_placeholder"

SECTION 9: QUICK CHECKLIST — COMPLETE SETUP SEQUENCE
================================================================================

  Run through this list in order before first use:

  [ ] 1. Get a Gemini API Key
          → https://aistudio.google.com/app/apikey
          → Paste into .env: GEMINI_API_KEY=AIza...

  [ ] 2. Copy .env.example to .env and fill in all values
          → copy .env.example .env
          → Set POSTGRES_PASSWORD (don't use "postgres" in production)
          → Set SECRET_KEY (run: python -c "import secrets; print(secrets.token_hex(32))")

  [ ] 3A. Start PostgreSQL (Docker path):
           → docker compose up db -d
           → Wait for health check to pass

  [ ] 3B. OR install PostgreSQL locally and create database:
           → CREATE DATABASE resume_intelligence;

  [ ] 4.  Activate backend virtual environment:
           → cd backend
           → python -m venv .venv
           → .venv\Scripts\activate  (Windows)

  [ ] 5.  Install Python dependencies:
           → pip install -r requirements.txt

  [ ] 6.  Pre-download spaCy model:
           → python -m spacy download en_core_web_sm

  [ ] 7.  Run database migrations:
           → alembic upgrade head

  [ ] 8.  Start ChromaDB (Docker path):
           → docker compose up chromadb -d

  [ ] 9.  Start the FastAPI backend:
           → uvicorn app.main:app --reload --host 0.0.0.0 --port 8080

  [ ] 10. Install frontend dependencies:
           → cd ../frontend
           → npm install

  [ ] 11. Fix local dev proxy (if not using Docker):
           → Edit frontend/vite.config.ts Line 18
           → Change target from 'http://backend:8080' to 'http://localhost:8080'

  [ ] 12. Start the frontend dev server:
           → npm run dev

  [ ] 13. Open: http://localhost:5173

  [ ] 14. Test API: http://localhost:8080/docs  (Swagger UI)

  [ ] 15. For full Docker deploy:
           → Add GEMINI_API_KEY=${GEMINI_API_KEY} to docker-compose.yml backend env
           → docker compose up --build
